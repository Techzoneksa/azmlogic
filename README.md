# منصة عزم للتشغيل اللوجستي

تجربة المرحلة الأولى لمنصة تشغيل لوجستي عربية بالكامل لإدارة الطلبات والطرود والشركاء والمناديب والمركبات والتوزيع ووثائق النقل وجاهزية الربط المستقبلي.

## التقنية

- Next.js App Router
- React
- TypeScript
- CSS مخصص بتخطيط RTL
- حالة عرض محلية بدون اعتماد على خادم خارجي في المرحلة الأولى
- أيقونات `lucide-react`

## الأوامر

```bash
npm install
npm run dev
npm run lint
npm run build
npm run start
```

## الأدوار التجريبية

- المدير العام
- مدير العمليات
- منسق التشغيل
- مسؤول الامتثال
- المندوب
- مشاهد التقارير

## المسارات الرئيسية

تشمل التجربة مسارات تسجيل الدخول ولوحة التحكم والشركاء والطلبات والطرود والتوزيع والمناديب والمركبات ونقاط الاستلام ومحاولات التسليم والمرتجعات ووثائق النقل وجاهزية بيان والتقارير وسجل الأنشطة والإعدادات وواجهة المندوب.

## جاهزية الإنتاج

- الواجهة عربية بالكامل واتجاهها RTL.
- ملفات التطبيق والأصول داخل المستودع.
- متغيرات البيئة موثقة في `.env.example`.
- لا يتم تضمين أسرار أو مفاتيح اعتماد داخل الواجهة.
- وثائق النقل في العرض محفوظة كمسودات ولا تمثل اعتمادا رسميا.
- المشروع مهيأ للبناء والتشغيل عبر أوامر npm القياسية.

## متغيرات البيئة

راجع `.env.example` وملف النشر في `docs/HOSTINGER_DEPLOYMENT.md`.

## Phase 1.2

تمت إضافة وحدة ملفات المناديب الرئيسية ووحدة المناطق والتغطية كواجهة تجريبية محلية. تشمل المرحلة بيانات الهوية والرخصة والعقد ونوع الاتفاق ونسبة الجاهزية وربط المندوب بالمناطق، مع صفحات مناطق التغطية ومؤشرات لوحة التحكم وفلاتر التوزيع. لا يوجد Backend أو قاعدة بيانات أو خرائط أو ربط بيان فعلي في هذه المرحلة.
## Phase 2 — تأسيس الباكند وقاعدة البيانات

تمت إضافة أساس إنتاجي للباكند دون إزالة وضع العرض التجريبي:

- Prisma ORM مع PostgreSQL.
- schema أولي يغطي المستخدمين والأدوار والصلاحيات والشركاء والمناديب والمركبات ومناطق التغطية والطلبات والطرود والتوزيع ومحاولات التسليم والمرتجعات ووثائق النقل وجاهزية بيان وسجل النشاط.
- طبقة خدمات server-side تعمل بقاعدة البيانات عند توفر `DATABASE_URL` وتعود إلى demo data عند غيابها.
- API foundation تحت `/api/foundation`.
- seed script لنسخ بيانات العرض الحالية إلى قاعدة البيانات.
- توثيق Hostinger وقاعدة البيانات في `docs/HOSTINGER_DEPLOYMENT.md` و`docs/DATABASE_SETUP.md`.

أوامر قاعدة البيانات:

```bash
npm run db:generate
npm run db:push
npm run db:seed
npm run db:studio
```

متغيرات البيئة المطلوبة للإنتاج:

```env
DATABASE_URL=
DIRECT_URL=
JWT_SECRET=
NEXT_PUBLIC_APP_URL=
NEXT_PUBLIC_DATA_MODE=database
```

عند غياب `DATABASE_URL` لا يتعطل التطبيق، ويظهر وضع البيانات التجريبي في صفحة الإعدادات.
